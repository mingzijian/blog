HTTP
--------

**Hyper Text Transfer Protocol**（超文本传输协议）。	
是用于从WWW服务器传输超文本到本地浏览器的传送协议。
HTTP是一个应用层协议，由请求和响应构成，是一个标准的客户端服务器(B/S)模型。
HTTP是一个无状态的协议。

#### HTTP响应状态码

消息响应/成功响应/重定向/客户端错误/服务器端错误
##### 消息响应

| 状态码 | 短语                              | 描述                                                         | 版本支持 |
| ------ | --------------------------------- | ------------------------------------------------------------ | -------- |
| 100    | Continue<br/>(继续)               | 客户端应当继续发送请求。                                     |          |
| 101    | Switching Protocol<br/>(切换协议) | 服务器已经理解了客户端的请求，并将通过Upgrade 消息头通知客户端采用不同的协议来完成这个请求。 |          |
| 102    |                                   | 由WebDAV（RFC 2518）扩展的状态码，代表处理将被继续执行。     |          |

##### 成功响应

| 状态码 | 短语                                           | 描述                                                         | 版本支持 |
| ------ | ---------------------------------------------- | ------------------------------------------------------------ | -------- |
| 200    | OK<br/>(成功)                                  | 请求已成功，请求所希望的响应头或数据体将随此响应返回。       |          |
| 201    | Created<br/>(已创建)                           | 请求已经被实现，而且有一个新的资源已经依据请求的需要而建立，且其 URI 已经随Location 头信息返回。<br/>假如需要的资源无法及时建立的话，应当返回 '202 Accepted'。 |          |
| 202    | Accepted<br/>(已接受)                          | 服务器已接受请求，但尚未处理。                               |          |
| 203    | Non-Authoritative Information<br/>(信息未授权) | 非权威性信息，服务器已成功处理了请求，但返回的实体头部元信息不是在原始服务器上有效的确定集合，而是来自本地或者第三方的拷贝。<br/>当前的信息可能是原始版本的子集或者超集。 |          |
| 204    | No Content<br/>(无内容)                        | 服务器成功处理了请求，但不需要返回任何实体内容，并且希望返回更新了的元信息。 |          |
| 205    | Reset Content<br/>(重置内容)                   | 服务器成功处理了请求，且没有返回任何内容。<br/>但是与204响应不同，返回此状态码的响应要求请求者重置文档视图。<br/>该响应主要是被用于接受用户输入后，立即重置表单，以便用户能够轻松地开始另一次输入。 |          |
| 206    | Partial Content<br/>(部分内容)                 | 服务器已经成功处理了部分 GET 请求。                          |          |



##### 重定向

| 状态码 | 短语                                | 描述                                                         | 版本支持 |
| ------ | ----------------------------------- | ------------------------------------------------------------ | -------- |
| 300    |                                     |                                                              |          |
| 301    |                                     | 永久性转移。<br/>被请求的资源已永久移动到新位置，并且将来任何对此资源的引用都应该使用本响应返回的若干个 URI 之一。 |          |
| 302    |                                     | 暂时性转移。<br/>请求的资源现在临时从不同的 URI响应请求。    |          |
| 303    |                                     | 对应当前请求的响应可以在另一个 URI 上被找到，而且客户端应当采用 GET 的方式访问那个资源。 |          |
| 304    | Not Modified<br/>(未修改)           | 304状态码或许不应该认为是一种错误，而是对客户端有缓存情况下服务端的一种响应。<br/>如果客户端发送了一个带条件的 GET 请求且该请求已被允许，而文档的内容（自上次访问以来或者根据请求的条件）并没有改变，则服务器应当返回这个状态码。 |          |
| 305    | Use Proxy<br/>(使用代理)            | 请求者只能使用代理访问请求的网页。                           |          |
| 306    | -                                   | 在最新版的规范中，306状态码已经不再被使用。                  |          |
| 307    | Temporary Redirect<br/>(临时重定向) | 请求的资源现在临时从不同的URI 响应请求。                     |          |

| 状态码 | 短语                                        | 描述                                                         | 版本支持 |

| 401    | 访问被拒绝                                  | 当前请求需要用户验证。<br/>401.1 - 未授权：登录失败。<br/>401.2 - 未授权：服务器配置问题导致登录失败。<br/>401.3 - 由于 ACL 对资源的限制而未获得授权。<br/>401.4 - 未授权：授权被筛选器拒绝。<br/>401.5 - 未授权：ISAPI/CGI 应用程序授权失败。<br/>401.7 – 访问被 Web 服务器上的 URL 授权策略拒绝。这个错误代码为 IIS 6.0 所专用。<br/><br/> |          |

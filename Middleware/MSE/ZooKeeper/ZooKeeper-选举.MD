ZooKeeper 选举
------------------------

### 选举算法

| 算法(机制)                | electionAlg | 说明 |
| ------------------------- | ----------- | ---- |
| LeaderElection            | 1           |      |
| AuthFastLeaderElection    | 2           |      |
| FastLeaderElection (默认) | 3           |      |



### 服务器角色

| 角色            | 提供服务 | 说明                               |
| --------------- | -------- | ---------------------------------- |
| Leader 领导者   | 读/写    |                                    |
| Follower 跟随者 | 读       |                                    |
| Observer 观察者 | 读       | 已得出选举结果,不用再投票,看着就行 |



### 选举状态

| 状态      | 定义     | 说明                                                       |
| --------- | -------- | ---------------------------------------------------------- |
| LOOKING   | 无主状态 | 处于该状态的节点,认为当前集群中尚无主节点Leader,会发起选举 |
| FOLLOWING | 跟随状态 | 表明当前节点的角色为Follower,已知Leader不是自己            |
| LEADING   | 领导状态 | 表明当前节点的角色为Leader,已知Leader是自己                |
| OBSERVING | 观察状态 | 表明当前节点的角色为Observer,已知Leader,可能是自己         |



#### LOOKING 寻找

众生平等,群龙无首
当前服务器不知道谁是Leader
竞选状态,执行选举

#### FOLLOWING 跟随

马首是瞻,忠心耿耿
当前服务器是Follower
同步Leader状态

#### LEADING 领导

号令天下,唯我独尊
当前服务器是Leader

#### OBSERVING 观察

登基大典,天位既定
投票结束,已选出Leader
